# Final submission checklist

Every box is a hard gate. Do not submit with assumptions, draft markers, or evidence links that only the owner can open.

## Product

- [ ] Glyph runs as described from a clean clone.
- [ ] Root README contains exact install, environment, run, and test instructions.
- [ ] Plugin/dev-tool install instructions and a test method that does not require rebuilding are verified.
- [ ] Final demo data is deterministic, legal to show, and free of secrets.
- [ ] All shipped claims in `DEVPOST.md` match the build; roadmap features are clearly labeled.
- [ ] A team member reviewed and understands every submitted AI-assisted code path.
- [ ] Working increments were committed often enough to make the final build recoverable.

## Video

- [ ] Final render is under three minutes.
- [ ] A complete exported-file watch confirms clear audio and readable visuals.
- [ ] Spoken voiceover explicitly covers concrete Codex and GPT-5.6 use.
- [ ] The real product, evidence behavior, and strongest end-to-end flow are visible.
- [ ] YouTube video is Public or Unlisted and works while signed out.
- [ ] Canonical YouTube link is pasted into Devpost and `EVIDENCE.md`.

## Codex and README

- [ ] `/feedback` was run from the main build thread in Codex Desktop or CLI opened on this codebase.
- [ ] The `/feedback` Session ID is entered in Devpost and `EVIDENCE.md`.
- [ ] README explains how Codex and GPT-5.6 were actually used with specific examples.
- [ ] At least one concrete part of the project used GPT-5.6; the submission does not falsely imply the whole build used it.
- [ ] README and Devpost contain no generic or unverifiable AI claims.

## Repository and team

- [ ] Canonical repository URL and final commit SHA are recorded.
- [ ] Repository visibility is verified.
- [ ] If Private, both `testing@devpost.com` and `build-week-event@openai.com` have the required access.
- [ ] Access is verified rather than inferred from an invitation attempt.
- [ ] If private-repository access is still pending, a deadline reminder is set.
- [ ] Every team member is added and has accepted the invitation.

## Devpost

- [ ] Final description was read aloud and rewritten where it sounded unnatural or inflated.
- [ ] A team member—not AI—selected and approved the project name.
- [ ] Project, video, repository, team, technologies, and session ID fields are complete.
- [ ] `./scripts/submission-audit.sh` reports PASS.
- [ ] Submission was sent before 5:00 PM PT with upload/render buffer.
- [ ] My Projects shows the entry in green as **Submitted**, not Draft.
