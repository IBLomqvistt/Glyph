# Glyph Prioritized TODOs

Last updated: 2026-07-21

## How to use this backlog

- Work from top to bottom within each priority unless a dependency is unresolved.
- Move a task to In progress only when its owner is actively working on it.
- A task is complete only when its acceptance check passes and evidence is linked from `docs/PROJECT_STATUS.md`.
- Tasks marked **Backend handoff** must be delegated by the PM to `lead_backend_engineer` once their dependencies are satisfied.
- Completing a task does not automatically complete its module; the module Definition of Done still governs acceptance.
